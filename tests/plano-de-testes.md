Plano de Testes: TechFuture Store
1. Objetivo dos Testes

Garantir que as funcionalidades principais do e-commerce (Cadastro, Login, Busca e Contato) estejam operando conforme o esperado, assegurando uma experiência de usuário sem erros críticos e validando a integridade dos dados inseridos.
2. Escopo

Os testes abrangerão as seguintes áreas do site:

    Funcional: Validação do fluxo de cadastro e login.

    Usabilidade: Verificação da facilidade de navegação e busca de produtos.

    Interface: Garantia de que o formulário de contato envia as informações corretamente.

3. Tipos de Teste Aplicados

Serão utilizados os seguintes tipos de teste:

    Testes Funcionais (Caixa Preta): Para validar se as entradas (inputs) geram as saídas esperadas.

    Testes de Regressão: Para garantir que novas alterações não quebrem funcionalidades existentes.

    Testes de Interface de Usuário (UI): Para verificar a disposição dos elementos e links de navegação.

4. Ambiente de Teste

    Navegadores: Google Chrome (versão mais recente) e Mozilla Firefox.

    Dispositivos: Desktop (Windows/MacOS) e dispositivos móveis (Android/iOS) para validar a responsividade.

    Ferramentas: Testes manuais realizados diretamente no navegador.

5. Critérios de Aceitação

O sistema será considerado apto para publicação se:

    100% dos casos de teste críticos (Login e Cadastro) passarem sem erros.

    O tempo de resposta da busca não exceder 3 segundos.

    Todas as mensagens de erro (ex: "Senha incorreta") forem exibidas de forma clara para o usuário.
